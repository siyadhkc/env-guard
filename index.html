Why env-guard
Without env-guard
code with secret
↓ commit
↓ push to GitHub
↓ GitHub scans
↓ alert sent
↓ key already exposed ✖
With env-guard
code with secret
↓ commit attempt
↓ blocked on your machine ✔
fix locally
↓ push clean code
↓ nothing leaked ✔

GitHub Secret Scanning is your last line of defense. env-guard is your first.
It runs locally, offline, before the secret ever leaves your machine.

What's new
v0.1.1
Custom rules.yml
Define your own secret patterns in a YAML file. Drop it in your project root or home directory — env-guard picks it up automatically.
v0.1.1
False positive fix
Tightened the Basic Auth URL pattern. Google Fonts and CDN URLs no longer trigger false HIGH severity alerts.
v0.1.0
58 passing tests
Full test suite covering every major rule and scanner behavior. Positive and negative cases for each pattern.
v0.1.0
CI/CD publish pipeline
GitHub Actions workflow. Tag a release and PyPI publishes automatically via trusted publishing — no tokens stored anywhere.
rules.yml  ·  drop this in your project root 
# your own secret patterns, loaded automatically
rules:
  - name: "Internal API Key"
    pattern: "MYCOMPANY_[A-Z0-9]{32}"
    severity: HIGH

  - name: "Internal DB Password"
    pattern: "(?i)internal_db_pass\\s*=\\s*[\"'][^\"']+[\"']"
    severity: HIGH
